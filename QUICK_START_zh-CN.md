# MultiAnno 标注项目快速开始

[English](./QUICK_START.md) | [简体中文](./QUICK_START_zh-CN.md)

本文档面向只负责标注工作的用户，目标是说明如何从一批多视图图像开始，创建项目、完成视图设置、进入工作区标注，并导出最终数据。

如果你还没有安装和启动 MultiAnno，请先参考 [README_zh-CN.md](./README_zh-CN.md)。安装完成后，在项目根目录运行：

```bash
python app.py
```

默认情况下，浏览器会打开：

```text
http://127.0.0.1:5173
```

## 1. 推荐的完整流程

一个标准标注项目建议按下面顺序进行：

1. 准备图像文件和项目目录。
2. 启动 MultiAnno。
3. 创建项目。
4. 在“视图设置”中添加图像文件夹、配置视图和工作空间。
5. 在“视图对齐”中确认各视图的裁剪和对齐关系。
6. 在“标注管理器”中设置类别和属性。
7. 在主工作区逐个场景完成标注。
8. 检查标注数量、类别、形状类型和保存状态。
9. 在“导出数据”中导出需要的标注或完整数据集。

单视图项目可以跳过多视图对齐步骤；不使用 AI 的项目也可以完全跳过 AI 设置。

## 2. 准备数据

### 2.1 建议的目录结构

建议为每个标注项目单独建立一个目录，例如：

```text
my_project/
  images/
    rgb/
    nir/
    dsm/
  workspace/
  export/
  project_meta.json
```

各目录用途：

- `images/`：原始图像目录，通常不要在标注过程中修改。
- `workspace/`：MultiAnno 保存标注 JSON 的工作空间。
- `export/`：导出 YOLO、COCO、VOC、MultiAnno 等格式时使用的目标目录。
- `project_meta.json`：项目元数据文件，记录视图、对齐、工作空间和项目配置。

### 2.2 多视图图像命名

MultiAnno 会根据文件名识别同一个场景组。多视图数据最好使用一致的场景编号，并通过后缀区分视图。

例如同一个场景 `scene_0001` 有三个视图：

```text
images/rgb/scene_0001_rgb.tif
images/nir/scene_0001_nir.tif
images/dsm/scene_0001_dsm.tif
```

在“视图设置”中可配置：

- RGB 文件夹后缀：`_rgb`，扩展名：`.tif`
- NIR 文件夹后缀：`_nir`，扩展名：`.tif`
- DSM 文件夹后缀：`_dsm`，扩展名：`.tif`

这样系统会把它们识别为同一个 stem：

```text
scene_0001
```

请尽量保持每个视图目录中的文件数量、命名规则和场景编号一致。命名不一致会导致某些场景无法组成完整的多视图场景组。

## 3. 创建项目

1. 启动 MultiAnno。
2. 点击顶部菜单中的“创建项目”。
3. 输入项目名称。
4. 选择或输入 `project_meta.json` 的保存路径。
5. 确认后进入“视图设置”。

如果你已经有一个项目元数据文件，可以使用“加载项目”打开 `project_meta.json`，系统会恢复视图、工作空间、类别和已有标注。

## 4. 配置视图

在“视图设置”中依次完成“文件夹”“视图”“工作空间”三个步骤。

### 4.1 添加文件夹

1. 在“文件夹”步骤中添加一个或多个图像文件夹。
2. 为每个文件夹确认后缀名和扩展名。
3. 如果系统自动推断正确，可以直接使用自动结果。
4. 如果自动推断不符合你的命名规则，请手动修改后缀名和扩展名。

常见例子：

```text
文件名：scene_0001_rgb.tif
后缀名：_rgb
扩展名：.tif
stem：scene_0001
```

### 4.2 添加视图

1. 在“视图”步骤中为每个文件夹创建对应视图。
2. 为每个视图选择源文件夹。
3. 设置通道、颜色映射或 RGB 映射。
4. 选择一个主视图。

主视图是标注坐标的基准。一般建议选择最清晰、最常用于判断目标边界的视图，例如 RGB 或正射影像。

其他视图作为辅助视图，用于同步浏览、对比和辅助判断。

### 4.3 设置工作空间

工作空间是标注 JSON 的保存位置。

建议使用独立目录，例如：

```text
my_project/workspace/
```

注意事项：

- 如果不单独设置工作空间，系统可能使用主视图文件夹作为默认保存位置。
- 如果工作空间中已经存在标注 JSON，系统会认为这是已有标注数据，并可能锁定路径。
- 不建议把工作空间和导出目录设为同一个目录。

确认无误后，点击“确认 & 开始对齐”。

## 5. 完成视图对齐

如果项目只有一个视图，系统会提示可以跳过视图对齐，直接进入标注工作区。

如果项目有多个视图，需要在“视图对齐”中逐个确认辅助视图与主视图的关系。

### 5.1 对齐时需要关注什么

对齐阶段主要确认两类参数：

- 裁剪：辅助视图中哪些区域参与显示和标注联动。
- 位置与缩放：辅助视图如何贴合主视图。

实际操作时可以：

1. 切换到需要处理的辅助视图。
2. 使用“裁剪”调整辅助视图有效范围。
3. 使用“移动/缩放”调整辅助视图位置。
4. 通过透明度、横向卷帘或纵向卷帘检查是否贴合主视图。
5. 如果图像范围应与主视图一致，可以尝试“适配主视图”。
6. 点击“确认当前视图”。
7. 所有辅助视图都确认后，点击“完成所有对齐”。

对齐参数会写入项目元数据。之后重新加载项目、使用 AI 或导出时都会依赖这些参数。

## 6. 设置标注体系

进入工作区后，建议先打开“标注管理器”设置类别和属性。

### 6.1 类别

类别用于表示目标类型，例如：

```text
building
road
tree
vehicle
```

建议：

- 在正式标注前尽量确定类别名称。
- 类别名称保持简短、稳定、大小写一致。
- 为不同类别设置容易区分的颜色。

### 6.2 属性

属性用于记录类别之外的信息，例如：

```text
occlusion: none / partial / heavy
quality: clear / blurry
source: manual / ai
```

如果多个类别都需要同一个属性，可以设置为全局属性。完成类别和属性设置后，再开始大规模标注。

## 7. 开始标注

主工作区一般包含：

- 中央图像画布。
- 左侧标注工具栏。
- 右侧对象、类别、属性和视图面板。
- 场景组列表或切片场景列表。

### 7.1 标注一个场景

1. 在场景列表中选择一个 scene/stem。
2. 确认主视图和辅助视图加载正常。
3. 选择目标类别。
4. 选择标注工具，例如矩形框、多边形、点、线、圆、椭圆等。
5. 在主视图上绘制目标。
6. 在弹出的对象信息中确认类别、属性和备注。
7. 保存对象。
8. 完成当前场景后切换到下一个场景。

标注坐标以主视图为基准。辅助视图用于同步检查和辅助判断，通常不需要分别在每个视图重复绘制。

### 7.2 常用操作建议

- 使用选择工具编辑已有对象。
- 使用右侧对象列表快速定位、检查和修改对象。
- 对难例可设置 `difficult` 或填写备注。
- 使用图层管理调整辅助视图显示、透明度或卷帘效果。
- 切换场景前观察顶部保存状态，确认没有保存失败提示。

### 7.3 AI 辅助标注

AI 是可选功能。如果你没有安装 AI 依赖或没有加载模型，可以完全使用手工标注。

如果需要 AI：

1. 在设置中打开 AI 设置并加载模型。
2. 在 AI 面板中选择目标视图。
3. 选择输出类型，例如多边形或矩形框。
4. 使用全自动、文本提示、点提示或框提示生成候选结果。
5. 检查结果是否正确。
6. 只确认质量合格的结果，不合格的结果应删除或手动修正。

AI 结果建议都经过人工确认后再进入正式数据集。

## 8. 质检标注

导出前建议至少做一次检查。

### 8.1 项目级检查

在“标注管理器”中刷新统计，重点检查：

- 总对象数量是否符合预期。
- 各类别数量是否异常。
- 形状类型分布是否异常。
- 是否有未注册类别或拼写错误类别。
- 属性值是否遗漏或不一致。

### 8.2 场景级检查

抽查或逐个检查场景：

- 图像是否加载正确。
- 标注是否与目标边界贴合。
- 类别是否正确。
- bbox、多边形、点、线等形状类型是否符合任务要求。
- 辅助视图对齐是否仍然正确。
- 是否存在明显漏标、错标、重复标注。

如果发现大量对齐问题，请回到“视图对齐”重新确认参数，再继续标注或导出。

## 9. 导出数据

完成标注后，打开“导出数据”。

### 9.1 选择导出模式

常见选择：

- 仅导出标注：只导出 JSON、TXT、XML 等标注文件。
- 导出完整数据：导出标注和图像，适合直接训练或交付数据集。

### 9.2 选择任务和格式

根据下游需求选择格式，例如：

- MultiAnno：保留 MultiAnno 原生标注结构。
- YOLO：适合 YOLO 检测或分割训练。
- COCO：适合 COCO 风格检测、分割任务。
- VOC：适合 Pascal VOC 风格任务。

不同格式对形状类型的支持不同。导出前请确认选择的类别和形状类型与下游任务一致。

### 9.3 设置目标目录

建议导出到独立目录，例如：

```text
my_project/export/yolo_dataset/
```

如果目标目录已有内容，系统会提示是否覆盖。只有在确认目录中没有需要保留的文件时，才选择覆盖。

导出完成后，建议打开目标目录检查：

- 标注文件是否生成。
- 图像是否按预期复制或转换。
- 类别文件是否正确。
- train/val/test 分割是否符合预期。
- 导出报告中的数量是否合理。

## 10. 继续已有项目

下次继续标注时：

1. 启动 MultiAnno。
2. 点击“加载项目”。
3. 选择之前保存的 `project_meta.json`。
4. 等待系统加载工作空间中的已有标注。
5. 继续选择场景进行标注。

不要在项目进行中随意修改图像文件名、移动图像目录或删除工作空间 JSON。确实需要调整时，建议先备份项目目录。

## 11. 常见问题

### 11.1 视图设置后没有场景

通常是文件后缀、扩展名或命名规则不一致。检查每个视图目录中的文件名是否能得到相同 stem。

### 11.2 辅助视图与主视图对不上

回到“视图对齐”，重新调整裁剪、缩放和偏移。确认每个辅助视图都点击过“确认当前视图”。

### 11.3 标注似乎没有保存

检查顶部保存状态和工作空间路径。确认工作空间目录可写，并且没有被其他程序锁定。

### 11.4 导出结果为空或数量不对

检查导出时选择的类别、形状类型和任务格式。也可以先在“标注管理器”刷新统计，确认工作空间中确实有标注。

### 11.5 AI 面板不可用

AI 功能依赖额外环境和模型。基础标注模式不需要 AI，若只做手工标注可以忽略该提示。

## 12. 标注前检查清单

开始正式标注前，建议确认：

- 图像目录已备份或保持只读。
- 多视图文件名可以对应到同一个 stem。
- 已选择正确主视图。
- 工作空间路径独立且可写。
- 多视图项目已完成视图对齐。
- 类别和属性已经定义。
- 随机打开几个场景确认图像和辅助视图显示正常。

## 13. 导出前检查清单

导出前，建议确认：

- 顶部没有保存失败提示。
- 标注管理器统计已刷新。
- 类别名称没有拼写错误或重复变体。
- 抽查过关键场景的标注质量。
- 导出格式、类别、形状类型选择正确。
- 目标目录可以被覆盖，或已经选择新的空目录。
